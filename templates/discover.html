Glossary

Here is a glossary of technical terms you may find while using this website

Pipeline A pipeline is a term used across many industries to describe an organised plan, with a start and end date, involving a series of stages.
In this context, we are looking for satellite images between two dates, hopefully delivering results within specified intervals.
It is similar to a project timeline, with defined goals along the way to be met by specific dates.
AOI The AOI, or Area of Interest, is the determined location on the surface of the Earth, of which you hope to receive satellite images.
AOI coverage The AOI coverage refers to the percentage of the AOI which was selected which could be captured by satellite imaging technology.
All pipelines created on our platform gurantee that all images will have captured at least 50% of the AOI selected.
Archived imagery Imagery which has already been captured and downloaded by satellites. If your pipeline involves any intervals which will start and finish before today will deliver archived images. These can be delivered straight away.
Tasking Tasking refers to the future plan of accessing satellite images, in effect intervals which are scheduled for any time after today.
Satellite aquisition The API used on this platform uses the AOI, interval dates and other parameters selected to match up to the most suitable satellite.
Satellites follow a specific trajectory around the Earth, therefore the API cross references the selected AOI with any satellites orbiting above that AOI.
If the satellite orbits above the AOI within the prescribed interval, and can meet all other parameter requirements, this is successful satellite aquisition.
Tasking interval The interval is the length of time between two dates. There will be several intervals within the pipeline. If the interval is set to one day, the user wishes to capture an image of that AOI every day. The longer the interval, the higher chance an image can be found for that interval, as there is less restriction on when the image can be taken and by which satellite.
Output type There are many different kinds of images satellites can capture, for all kinds of purposes.
For example, infrared satellite imagery can be used for global warming projects, to capture temperature differences on the Earth's surface and in the atmosphere.
Metadata Metadata is considered 'data about data'; in this case, refers to the raw data about the satellite used to capture the image.
Any images delivered to your pipeline will provide a relevant link with all the metadata for that image. This data is used when post processing the image for whatever the purpose may be.
Cloud cover percentage This parameter refers to the maximum allowable percentage of the delivered image which can be obscured by clouds.
Image resolution This refers to the detail an image holds, and how many visible pixels proportionate to the size of the image. An image with a resolution of 10 metres means that each pixel stands for a 10 m x 10 m area on the ground.
Images available to the users on this platform will have a resolution between 8 and 15 metres.
